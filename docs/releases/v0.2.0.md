# v0.2.0

`v0.2.0` is the first release where the GWT claim is backed by a concrete
post-broadcast runtime path instead of only workspace selection and trace text.

## Highlights

- `SelectionBroadcastCycle` owns a post-broadcast `BroadcastBus`.
- Independent subscribers read the same broadcast and propose exact resolution,
  semantic recall, relation continuation, contradiction flags, or follow-up
  requests.
- `gwt_attend` applies accepted proposal kinds through public application ports.
- Workspace admission has ignition gating through `GWT_MIN_ACTIVATION`.
- Conscious workspace items reactivate explicit `gwt_link` targets into the
  preconscious buffer for recurrent cycles.
- `gwt_bus_inspect` and `gwt_trace_explain` expose accepted, inhibited, and
  executed bus actions.
- `ExternalReasoningSubscriber` provides the port-safe adapter path for future
  LLM/NLI/agent-loop subscribers.

## Known Limits

- Built-in subscribers are deterministic and local; provider-backed external
  subscribers are intentionally adapter-owned.
- Multi-pass attention remains opt-in through `gwt_attend(passes=...)`.
- Benchmarks show bus health and no regression in the bounded matrix, not a
  universal quality guarantee.
