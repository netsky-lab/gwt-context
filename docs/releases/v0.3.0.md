# v0.3.0

`v0.3.0` is the target release for making the GWT runtime usable as a Codex MCP
memory system rather than only a benchmark/demo runtime.

## Highlights

- MCP memory management surface:
  `gwt_memory_profile`, `gwt_reset`, `gwt_export_memory`, and
  `gwt_import_memory`.
- Structured runtime read models bootstrap from persisted memory after MCP
  startup, so exact `gwt_collection_query` survives restarts.
- Stored records receive active namespace tags for project/global separation.
- Broadcast-bus arbitration records decision reason codes, summaries, and
  proposal grouping in traces and inspect output.
- Optional OpenAI-compatible external subscribers can be wired at the MCP
  composition root with separate bus and provider budgets.
- Integration coverage proves recurrent linked-memory activation and accepted
  subscriber queries can affect later workspace contents.
- Codex bootstrap, health, and cleanup helpers support split project/global MCP
  namespaces.

## Verification

Run before tagging:

```bash
python scripts/release_gate.py
npm run memory:health -- --smoke
python scripts/qwen_sanity.py --run --max-tasks 1
python -m tests.benchmarks.bus_matrix --summarize tests/benchmarks/results/*.json
python -m tests.benchmarks.analyze_results tests/benchmarks/results/*.json
```

## Release Gates

- `pytest`, `ruff`, `mypy`, npm verification, stdio MCP smoke, examples, build,
  and boundary checks pass.
- No tracked `.env`, benchmark result artifacts, supervisor folders, or real
  RunPod URLs.
- Bus-on accuracy is not below bus-off on the same benchmark slice.
- Bus-on average tool calls are no more than `0.25` above bus-off.
- Bus subscriber timeout/error count is `0 / 0`.
- Benchmark summaries include bus inhibition reason counts.

## Known Limits

- External subscribers are opt-in and should be treated as experimental until a
  larger bus on/off matrix is regenerated.
- Persistent memory deletion remains a local maintenance script, not an MCP
  tool.
- The project/global split is convention-based through `GWT_DATA_DIR`; it is not
  a remote multi-tenant authorization layer.
